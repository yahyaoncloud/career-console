HormuzWatch Cloud Architecture

Last updated · Thu, 25 Jun 2026  /  v1.0  /  3 phases
A real-time intelligence platform built around continuous telemetry ingestion, low-latency event processing, and live geospatial queries.

HormuzWatch is a streaming intelligence system where data flows continuously from AIS, OpenSky, and satellite APIs through anomaly detection workers to a live operations dashboard. The architecture prioritizes low latency over throughput consistency and decouples ingestion from processing using an event-driven spine.

The most consequential decision in this architecture is not using PostgreSQL as a queue. For a platform ingesting thousands of position reports per second, a database-backed job queue would collapse under write load. Instead, Azure Event Hubs becomes the system spine, with each worker group subscribing independently via consumer groups.

Design priorities: low latency · continuous ingestion · live maps · event processing · AI inference · geospatial queries · high availability.

2 · Networking model

Azure Virtual Network
├── Container Apps Environment        (subnet /23, delegated)
├── VM Scale Set                      (subnet /24)
├── PostgreSQL                        (Private Endpoint)
├── Blob Storage                      (Private Endpoint)
├── Key Vault                         (Private Endpoint)
├── Event Hubs Namespace              (Private Endpoint)
└── Private Endpoint Subnet           (non-delegated /24)

Ingress:  only via Front Door Premium over Private Link
Egress:   ACR, Entra, AIS Stream, OpenSky, satellite APIs
NSG:      deny all inbound except Front Door service tag on 443
Why Front Door Premium. Standard Front Door cannot originate to Private Link. Premium is required to route traffic to an internal Container Apps environment through a private endpoint. The Container Apps environment has public network access disabled; the only way in is Front Door.

6 · When to introduce each service

VM Scale Set — when single VM CPU is consistently above 70% or consumer-group lag grows beyond acceptable thresholds.

Event Hubs Premium — when you need guaranteed throughput units, Kafka compatibility, or 90-day retention for audit.

Redis — when the dashboard's live map starts pinging PostgreSQL for last-known-position queries and p95 latency exceeds 200ms.

Geo-replicated PostgreSQL — when regional outage becomes unacceptable to the business, not before.

Microsoft Sentinel — when a customer or compliance framework requires SOC-grade log correlation and incident response.

AKS — only when independently deployable services make Container Apps' revision model painful. Don't add it for its own sake.

API Management — when you ship a partner-facing intelligence API or need fine-grained rate limiting per customer.
